# OpenWrt Pure Stable Auto-Build (x86_64)

[![Build OpenWrt & Release](https://github.com/sammy0101/Build-OpenWrt/actions/workflows/build-openwrt-pure.yml/badge.svg)](https://github.com/sammy0101/Build-OpenWrt/actions/workflows/build-openwrt-pure.yml)

這是一個基於 GitHub Actions 的自動化構建腳本，用於編譯 **OpenWrt 官方最新穩定版 (Pure Stable)** 固件。
無需本地搭建環境，只需在網頁上點擊即可生成自定義的 x86_64 固件。

This is an automated GitHub Actions workflow to build the **Official OpenWrt Latest Stable** firmware.
No local environment required. Just click to build your custom x86_64 firmware.

## ✨ 特點 / Features

*   **純淨原版 (Pure Source)**: 代碼直接拉取自 [OpenWrt 官方倉庫](https://github.com/openwrt/openwrt)。
*   **自動追新 (Auto Latest)**: 自動過濾並鎖定最新的 `vX.X.X` 穩定版 Tag (排除 RC 測試版)。
*   **高度客製 (Customizable)**: 支援自定義 **LAN IP** 和 **固件分區大小 (RootFS)**。
*   **包含 LuCI (Web UI)**: 預裝官方 LuCI 介面 (英文原版)，支援 HTTPS。
*   **極致壓縮 (Compressed)**: 自動生成 `.vmdk.gz` (VMware/ESXi) 和 `.img.gz` (Phys/PVE)，體積小，下載快。
*   **依賴修復 (Fixes)**: 內建修復 `qemu-utils` 依賴，確保 VMDK 格式正確生成。

## 🚀 如何使用 / How to Use

1.  **Fork** 本倉庫到你的 GitHub 帳號。
2.  進入項目的 **Actions** 頁籤。
3.  在左側選擇 **Build OpenWrt & Release**。
4.  點擊右側的 **Run workflow** 按鈕。
5.  輸入你想要的配置：
    *   `LAN IP`: 路由器的管理地址 (例如 `192.168.10.1`)。
    *   `RootFS Size`: 固件大小 (MB) (例如 `2048`)。
6.  點擊綠色的 **Run workflow** 開始編譯。

等待編譯完成後（通常約 15-20 分鐘），請前往 **Releases** 頁面下載。

## 📂 檔案說明 / File Descriptions

編譯完成後，你會在 Releases 看到以下幾種檔案：

1.  **`generic-ext4-combined-efi.img.gz`** (推薦 / Recommended)
    *   適用於：實體機、PVE (Proxmox VE)、UNRAID。
    *   啟動方式：**UEFI** 引導。
    *   使用方法：解壓後寫入硬碟，或在 PVE 中使用 `qm importdisk`。

2.  **`generic-ext4-combined.img.gz`**
    *   適用於：較舊的實體機。
    *   啟動方式：**Legacy BIOS (CSM)** 引導。

3.  **`generic-squashfs-combined-efi.img.gz`**
    *   OpenWrt 預設格式，支援系統重置 (Factory Reset)，但在 x86 上擴容較麻煩。通常建議使用 ext4 版本。

4.  **`generic.vmdk.gz`** (虛擬機專用 / For VM)
    *   適用於：**VMware Workstation**、**ESXi**。
    *   使用方法：解壓後獲得 `.vmdk` 文件，建立虛擬機時選擇「使用現有虛擬磁碟」即可。
